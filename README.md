### 📝 임베디드 프로젝트 C 코딩 컨벤션 (MISRA C 2012 기반)

#### 1. 변수와 데이터 타입 (Data Types & Variables)
* **크기가 명확한 `stdint.h` 자료형을 사용한다.**
    * `int`, `char`, `short` 대신 시스템에 독립적인 `uint8_t`, `int32_t`, `uint16_t` 등을 사용하여 메모리 크기를 명확히 합니다.
* **변수는 선언과 동시에 반드시 초기화한다.**
    * 쓰레기값(Garbage value)으로 인한 오작동을 막기 위해 항상 기본값을 할당합니다.
    * *예시:* `uint16_t sensor_value = 0U;`
* **상수(숫자)를 쓸 때는 데이터 타입 접미사를 명시한다.**
    * 부호 없는 정수에는 `U`, 부동소수점에는 `F`를 붙여 컴파일러가 헷갈리지 않게 합니다.
    * *예시:* `0` 대신 `0U`, `3.14` 대신 `3.14F`

#### 2. 제어문 흐름 (Control Flow)
* **`if`, `for`, `while` 뒤에는 단 한 줄의 코드라도 반드시 중괄호 `{ }`를 씌운다.**
    * 나중에 코드를 추가하다가 발생하는 엉뚱한 버그를 원천 차단합니다.
* **`if - else if` 구조의 마지막은 반드시 `else { }` 로 닫아준다.**
    * 논리적으로 예외가 없다고 생각되더라도 방어 코드를 넣습니다. 할 일이 없다면 주석이라도 남깁니다.
    * *예시:* `else { /* 예상치 못한 상태, 아무 동작 안 함 */ }`
* **`switch` 문에는 반드시 `default:` 구문을 넣고, 각 `case`는 `break;`로 끝낸다.**

#### 3. 함수 작성 (Functions)
* **파라미터가 없는 함수는 괄호 안에 `void`를 명시한다.**
    * 빈 괄호 `()`는 C 언어에서 '아무 인자나 받을 수 있다'는 뜻으로 해석될 수 있습니다.
    * *예시:* `void init_system(void);`
* **함수 내부에서 값을 바꾸지 않는 읽기 전용 포인터 파라미터에는 반드시 `const`를 붙인다.**
    * 데이터 원본이 훼손되지 않음을 컴파일러와 팀원 모두에게 보장합니다.
    * *예시:* `void transmit_data(const uint8_t *p_data, uint16_t length);`
* **함수의 반환값(Return value)은 항상 변수에 받아서 확인하거나 처리한다.**
    * 성공/실패 여부를 반환하는 함수를 호출만 하고 무시하지 않습니다.

#### 4. 메모리 및 안전성 (Memory & Safety)
* **동적 할당(`malloc`, `free`) 대신 크기가 고정된 정적 배열(Static Array)을 사용한다.**
    * 런타임 중 메모리 누수나 단편화로 인해 시스템이 멈추는 것을 막기 위해, 필요한 버퍼는 프로그램 시작 시점에 전역/정적으로 확보합니다.
    * *예시:* `uint8_t rx_buffer[MAX_BUFFER_SIZE] = {0U};`
* **다른 타입이나 크기의 변수끼리 연산/대입할 때는 괄호로 명시적 형변환(Casting)을 한다.**
    * 컴파일러가 알아서 바꿔주는 암시적 형변환을 믿지 않습니다.
    * *예시:* `uint32_t total = (uint32_t)value1 + (uint32_t)value2;`

#### 5. 매크로 및 상수 (Macros & Constants)
* **복잡한 연산이 필요한 경우 `#define` 매크로 대신 `inline` 함수를 사용한다.**
    * 매크로 함수는 타입 검사를 하지 않아 괄호 실수 등으로 인한 버그가 잦습니다.
* **단순한 매직 넘버(의미를 알 수 없는 숫자) 대신 상태값은 `enum`을 활용한다.**
    * 코드의 가독성을 높이고 허용되지 않은 값이 들어가는 것을 막습니다.
